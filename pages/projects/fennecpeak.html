The Combat System

The game is focused around the combat and snappy movement so combat had to feel right. In the beginning we tried using colliders and triggers but realised it did not feel good enough. To make the combat feel better we instead used raycasts and since the characters are swinging their weapons the rays had to go in an arc.

One problem we had with this was that the rays shot out from the characters middle point going straight forward. This was a problem when we introduced height, jumping and smaller objects into the game!

To fix this problem we used 3 rays with different starting heights on the character, one on the head, the feet and the middle of the character.


//TOP ATTACK int angle = (int)(hitArcAngle / 2) * -1; 
rayNumber = (int)hitArcAngle / 5; 
for (int i = 0; i < rayNumber; i++) 
{ 
    //Calculate ray direction 
    Vector3 forwardDirection = Quaternion.Euler(0, angle, 0) * transform.forward; 
    angle += 5; 
    //Ray origin 
    Vector3 topPos = new Vector3(transform.position.x, transform.position.y + 1.5f, transform.position.z); 
    
    RaycastHit[] hits = Physics.RaycastAll(topPos, forwardDirection, maxAttackLength); 
    for (int j = 0; j < hits.Length; j++) 
    { 
        if (!hits[j].collider.isTrigger) 
        { 
            IKnockback knockbackEffect = hits[j].collider.GetComponent<IKnockback>();
            if (knockbackEffect != null) 
            { 
                Vector3 knockback = transform.forward * knockBackForce * attackCharge; 
                knockbackEffect.Knockback(gameObject, knockback); 
            } 
            break; 
        } 
    } 
    //Debug lines to see the rays
    Debug.DrawLine(topPos, topPos + forwardDirection * maxAttackLength, Color.red, 2); 
}

This was the solution that made the most sense with our time and experience in mind. Why complicate things when this worked well for our game?

Power-up

The power-up structure is something that I am not happy with how it turned out.

The structure is currently that the power-up spawner handles the timer for spawning the mesh and collider, The power-up itself handles the collision and when the player steps within the collision of the power-up it has to search the player for 2 components located on different child objects. The characters weapon shader and the script with the weapons variables

The structure I want it to be is that the player stores the variables and the shader on the main object for the player, making it easier to get the components needed. The power-up could handle both spawning timer and the collision by itself reducing the amount of objects involved in the sequence.
